# 수업 실습 따라하기

> ⚠️ GitHub는 코드를 **실행**하는 곳이 아니라 **저장**하는 곳이에요!

---

## 실습 A — 첫 커밋

1. [github.com](https://github.com) 접속 후 로그인
2. 우측 상단 `+` → `New repository`
3. Repository 이름 입력 (영어, 예: `my-first-repo`)
4. **`Add a README file` ✅ 체크** ← 꼭 체크!
5. `Create repository` 클릭
6. `Add file` → `Create new file`
7. 파일명: `hello.py`
8. 아래 코드 입력:

```python
# 나의 첫 번째 Python 파일
print("안녕하세요! 첫 커밋을 축하합니다 🎉")
print("GitHub는 코드를 저장하는 곳이에요.")
```

9. 오른쪽 위 `Commit changes` 클릭
10. 커밋 메시지 입력 (예: `첫 번째 커밋`)
11. `Commit changes` 클릭
12. 시계 아이콘(History) 클릭 → 커밋 히스토리 확인 ✅

---

## Branch 만들어보기

1. 내 저장소에서 파일 목록 위쪽 `main ▼` 버튼 클릭
2. 새 브랜치 이름 입력 (예: `my-feature`) → `Create branch`
3. 브랜치가 `my-feature`로 바뀐 걸 확인 → 파일 수정 후 Commit
4. 다시 `main ▼` → `main`으로 돌아오기
5. **파일이 수정 전 상태로 돌아온 걸 확인!** ← 이게 Branch의 핵심
6. `Compare & pull request` → Merge 해보기

## README.md 꾸미기

내 저장소에서 `Add file` → `Create new file` → 파일명: `README.md`

```markdown
# 나의 첫 번째 프로젝트

## 소개
이 프로젝트는 ...

**Python** 으로 만들었어요

> 한 줄 강조

- 기능 1
- 기능 2
```

오른쪽 `Preview` 탭으로 실시간 확인 가능!

## 실습 B — Fork & Pull Request

1. [github.com/gbshs/github-lecture-practice](https://github.com/gbshs/glp2) 접속
2. 우측 상단 `Fork` 클릭 → 내 계정으로 복사
3. 내 Fork 저장소에서 `Add file` → `Create new file`
4. 파일명: `contributors/내이름.md` (예: `contributors/김북곽.md`)
   - `contributors/` 를 앞에 붙이면 자동으로 폴더가 생겨요!
5. 아래 내용 입력:

```markdown
# 김북곽

한마디: "오픈소스의 거인이 될 거예요!"
```

6. `Commit changes`로 저장
7. `Contribute` → `Open Pull Request` → `Create Pull Request` 
8. 선생님이 merge하면 내 이름이 공식 저장소에 올라가요 

>  **왜 파일을 따로 만드나요?**
> 같은 파일을 여러 명이 동시에 수정하면 충돌(Conflict)이 생겨요.
> 각자 다른 파일을 만들면 충돌 없이 모두의 PR이 깔끔하게 merge됩니다.

## GitHub Pages — 내 코드가 웹사이트로!

1. 내 저장소 → 상단 `Settings` 클릭
2. 왼쪽 사이드바에서 `Pages` 클릭
3. Source → `Deploy from a branch` → `main` 선택 → `Save`
4. 저장소에 `index.html` 파일 만들기:

```html
<html><head> 김북곽의 Github Page </head>
<body>
<h1>안녕하세요! 저는 김북곽입니다.</h1>
<p>GitHub로 첫 웹페이지를 만들었어요 🎉</p>
</body>
</html>
```

5. 잠깐 기다리면 아래 주소로 웹사이트가 생겨요!

```
https://내아이디.github.io/저장소이름
```
